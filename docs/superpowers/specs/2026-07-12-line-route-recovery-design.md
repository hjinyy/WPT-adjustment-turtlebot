# 라인 중심 주행과 마커 기반 회전 복구 설계

## 목적

전역 AprilTag 지도를 이용해 코일 사이를 이동하되, 검정 테이프가 설치된
마커 없는 구간에서는 카메라 중심과 라인 중심을 일치시키며 주행한다. 라인
관측과 전역 위치추정을 분리해 바퀴 같은 검은 물체를 라인으로 잘못 인식해도
주행 제어가 흔들리지 않게 한다.

이 설계는 모든 코일 경로에 적용한다. `coil_4 -> coil_3`의 출발 방향은
서쪽이며, 출발 마커는 `43`, 목표 확인 마커는 `33`이다.

## 실제 로그에서 확인한 문제

2026-07-11 실제 주행 로그 두 건에서 다음을 확인했다.

- 첫 실행은 `42`만 24프레임 검출한 뒤 마커가 사라졌고,
  `ROTATE_TO_ROUTE -> LOCALIZE -> ERROR`로 전환됐다. 기존 전역 자세 예측
  허용 시간이 0.4초이고, 마커 손실 정지 시간이 2초이기 때문이다.
- 두 번째 실행은 `44`를 48프레임 검출했다. 마커가 보일 때는 `-0.18rad/s`로
  회전하고, 사라지면 `LOCALIZE`의 `+0.10rad/s` 탐색으로 바뀌었다. 따라서
  회전 방향이 반복해서 뒤집혔다.
- 두 실행의 라인 관측은 각각 325, 370프레임이며 각도가 거의 `-90~+90도`였다.
  실제 320x240 프레임의 최대 검은 윤곽은 바퀴 영역(약 19,267px)이었다.
  테이프가 아니라 바퀴를 라인으로 오인식한 것이다.

## 카메라와 라인 모듈

라인 검출은 전방 카메라만 사용한다. 화면 상단이 진행 방향이고 하단은 로봇
몸체 및 바퀴가 차지하므로, 라인 모듈에만 다음 ROI를 적용한다.

- 세로: 화면 상단부터 몸체가 나타나기 전까지. 초기값은 `0~55%`다.
- 가로: 카메라 중심을 포함한 중앙 폭. 초기값은 `20~80%`다.
- AprilTag 전역 위치추정과 최종 코일 정합에는 ROI를 적용하지 않고 전체 화면을
  사용한다.

검출기는 ROI 안의 어두운 윤곽 중 세로 방향이고 충분히 긴 후보만 선택한다.
큰 가로 윤곽, ROI 경계와 접한 넓은 윤곽, AprilTag의 작은 내부 무늬는 제외한다.
라인의 중심 `x`와 이미지 중심 `width/2`의 차이를 횡방향 오차로 쓴다.

## 제어와 필터 역할

라인 주행에는 칼만 필터를 사용하지 않는다. 매 프레임의 라인 중심 오차를 주
제어값으로 하고, 라인 기울기를 작은 보조 제어값으로 더하는 직접 PD 조향을
사용한다. 선이 화면 중앙에 오면 각속도는 0에 수렴한다.

EKF는 AprilTag의 전역 자세 측정과 `/odom` 예측에만 사용한다. 최종 코일 정합은
기존 AprilTag 쌍 중심 오차의 칼만 필터와 언더슈트 우선 정지를 유지한다.

## 상태기계

### IDLE

0 속도를 발행하고 명시적인 시작 요청만 기다린다.

### ACQUIRE_ROUTE

현재 코일과 목표 코일의 전역 좌표로 이동 방위를 정한다. 출발 코일의 해당
방위 마커가 화면 중심 전방에 들어와야만 전진을 허용한다.

`coil_4 -> coil_3`은 서쪽이므로 `43`을 기다린다. 마커가 한 프레임 사라져도
전진하지 않고, 마지막 유효 회전 부호와 `/odom`의 회전량을 이용해 같은 방향으로
계속 회전한다. 이 상태에서 반대 방향 탐색 명령으로 바꾸지 않는다.

### FOLLOW_LINE

출발 마커가 확인되면 라인 중심을 카메라 중심에 맞춰 전진한다. AprilTag가
보이면 EKF를 보정하고, 보이지 않으면 `/odom`으로 EKF를 예측한다. 라인이
검출되는 동안에는 마커가 없어도 주행한다.

라인과 마커가 동시에 사라진 경우 마지막 조향을 유지해 최대 0.5초만 전진한다.
그 안에 둘 중 하나가 복귀하면 계속 주행한다.

### REACQUIRE

0.5초 안에 라인과 마커를 모두 다시 얻지 못하면 선속도를 0으로 하고, 마지막
유효 조향 방향으로 제자리 회전한다. 라인 또는 마커를 찾으면 해당 상태로
복귀한다. 정지 후 후진으로 보정하지 않는다.

### APPROACH_TARGET 및 ALIGN_COIL

목표 이동 방향의 마커가 보이면 경로 주행을 끝낸다. `coil_4 -> coil_3`에서는
`33` 검출이 조건이다. 이후 전체 화면에서 `33/34` 쌍을 이용해 기존 시각 서보와
언더슈트 우선 최종 정합을 수행한다. 완료, 오류, 정지 요청은 항상 0 속도를
발행한다.

## 안전 경계

- 전진은 출발 마커 확인 이후에만 허용한다.
- 라인과 마커가 모두 없는 상태에서 전진 유지 시간은 0.5초를 넘기지 않는다.
- 회전은 마커 소실만으로 중단하지 않되, 설정된 탐색 시간 안에 라인 또는
  AprilTag를 재인식하지 못하면 ERROR와 0 속도로 전환한다.
- 목표 마커 없이 좌표만으로 최종 정합 상태로 전환하지 않는다.

## 성능 진단

매 제어 주기마다 다음 시간을 로그에 기록한다.

- 카메라 프레임 취득 시간
- AprilTag 검출 시간
- 라인 ROI/윤곽 검출 시간
- 전체 제어 주기 시간과 10Hz 주기 초과 여부

실제 라즈베리파이에서 30초 dry-run을 실행해 제어 주기 시간의 p95가 100ms 이하,
p99가 150ms 이하인지 확인한다. 기준을 넘으면 해상도, AprilTag 검출 빈도, 또는
제어 주기를 조정하되 안전 상태 전이는 바꾸지 않는다.

## 검증

- 가로 바퀴 윤곽은 라인으로 거부하고, 중앙 세로선은 검출하는 단위 테스트
- `coil_4 -> coil_3`에서 `43` 전에는 선속도 0, `43` 중앙 확인 뒤 전진 허용,
  `33` 검출 뒤 최종 정합 전환 테스트
- 마커가 사라져도 회전 부호가 유지되는 상태기계 테스트
- 라인/마커 동시 소실 0.5초 유예와 REACQUIRE 전환 테스트
- 기존 전역 지도, EKF, 최종 언더슈트 정합 테스트
- 실제 파이에서 dry-run 성능 로그와 상단 중앙 ROI 프레임 확인
